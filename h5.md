# h5 | Embedded devices

## a)

First we download the firmware with the AWS-CLI tool using the command `aws s3 ls s3://download.tplinkcloud.com/ --no-sign-request --recursive | grep Tapo_C200v3_en_1.4.2_Build_250313_Rel.40499n_up_boot-signed_1747894968535.bin`

Then we download and binwalk and tp-link-decrypt by cloning it from the github repo using `git clone <github ur>)`. 

## b)

Moving on to getting the tp-link-decrypt open. I had to install a few dependencies (xxd, jefferson, ubi-reader and openssl-devel) and change the ./preinstall.sh source code to support my Fedora distro.

This did the trick. I was able to run both extract_keys.sh and preinstall.sh! After this I ran the `make` command and it created a bin directory with a runnable tp-link-decrypt program!

With this program, I decrypted the previously installed firmware and got the decrypted version! This contains the file sqashfs-root, which contains bin, config, etc, lib and usr directories all decrypted and accessible!

## c) and d)

Now that we have access to the firmware, we can open the main program in the bin directory in Ghidra and look at how it works! Since the camera is used with a web interface, we can try and peek into the main program to see what comes in and out, where passwords are stored and how we can possibly reverse-engineer the firmware.
